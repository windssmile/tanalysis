## 1. 项目目标

开发一个网页版应用，系统化介绍常用的 **K线 / 分时图股票技术分析**框架与知识。

- **目标用户**：有一定基础的中国大陆散户，已掌握基本概念，希望系统学习各种形态、指标与分析框架。
- **核心策略**：先搭好可持续扩展的**框架**，再循序渐进地填充内容。第一版只做"小而完整"的第一部分。
- **语言/习惯**：面向中国大陆股民，**红涨绿跌**（阳线/上涨=红，阴线/下跌=绿）。

### 内容总蓝图（6 大板块，分期建设）

| # | 板块 | 第一版 |
|---|------|--------|
| 1 | K线基础（单根/组合形态） | ✅ 启用 |
| 2 | 技术指标（MA/MACD/KDJ/RSI/BOLL…） | ✅ 启用 |
| 3 | 经典形态（头肩顶底、双顶底、三角形…） | 占位 |
| 4 | 趋势理论/框架（道氏、波浪、支撑阻力、缠论…） | 占位 |
| 5 | 分时图分析（分时线、均价线、量价、盘口） | 占位 |
| 6 | 实战框架（多指标组合成分析流程/交易体系） | 占位 |

6 大板块从第一天就在数据中成形（`enabled` 标志控制），后续开启即可。

## 2. 技术栈

- **框架**：React + Vite（纯前端单页应用，无后端）
- **路由**：react-router
- **图表**：SVG + React 自绘（不引第三方图表库）
- **部署**：纯静态，可挂 Vercel / GitHub Pages（第一版先本地跑通，部署按需）

选 SVG 自绘的理由：本应用画的是"教科书示意图"而非真实行情，需要对每根 K线、每处标注（高亮、区域框、箭头、文字）的完全控制；SVG 直接、轻量，且天然适配深色主题与红涨绿跌配色。

## 3. 架构（方案 A：数据驱动 + 图表组件注册表）

核心理念：**内容与框架彻底分离**。加一个新条目 = 在内容层加一个数据对象（必要时在图表层加一个图表组件并注册）；板块、导航、版式全部自动生成。

```
React + Vite 纯前端单页应用
│
├── 内容层 (content/)          ← "填内容"只改这里
│   ├── topics/                  每个条目一个数据对象
│   └── categories.js            6 大板块定义（第一版启用前 2 个）
│
├── 渲染层 (components/)
│   ├── TopicPage                通用条目渲染器：吃一个 topic 对象 → 统一版式
│   ├── CategoryPage             板块目录页：列出该板块条目卡片
│   ├── HomePage                 首页：板块入口
│   └── layout/                  顶部导航、侧边栏、免责声明页脚
│
├── 图表层 (charts/)           ← 可交互 K线/指标示意图
│   ├── chartRegistry            id → 图表组件 的注册表
│   ├── CandleChart              通用 K线绘制组件（SVG）
│   └── indicators/              MA / MACD / KDJ 等指标绘制 + 计算
│
└── 路由 (react-router)
    /                            首页（板块入口）
    /:category                   板块目录页
    /:category/:topic            具体条目页
```

**数据流**：路由解析 `category/topic` → 内容层取 topic 对象 → `TopicPage` 渲染文字字段 → 遇 `chartId` 引用就去 `chartRegistry` 取组件渲染。

## 4. 内容数据结构

每个条目是一个结构化对象（示例：看涨吞没）：

```js
{
  id: "bullish-engulfing",
  category: "candlestick",        // 所属板块
  title: "看涨吞没",
  subtitle: "Bullish Engulfing",
  tags: ["反转形态", "看涨", "组合形态"],   // 渲染成胶囊标签
  chartId: "bullish-engulfing",   // 引用图表层的示意图 id
  sections: {
    meaning:    "由一阴一阳两根K线组成，第二根阳线实体完全包住前一阴线……",
    identify:   ["出现在下跌趋势末端", "第二根为阳线且实体吞没前阴线实体", "..."],
    usage:      ["可作为试探性买入信号", "结合成交量放大确认更可靠", "..."],
    limitation: "在震荡市中信号频繁且容易失效，需结合趋势与量能过滤。"
    // formula: 指标类条目可选，讲计算原理
  },
  related: ["bearish-engulfing", "hammer"]   // 关联条目，页尾"延伸阅读"
}
```

**统一字段约定**（所有条目遵守，渲染器据此自动排版）：

| 字段 | 含义 | 渲染形式 | 必填 |
|------|------|----------|------|
| `meaning` | 含义 | 段落 | 是 |
| `identify` | 识别要点 | 要点列表 | 是 |
| `usage` | 使用提示 | 要点列表 | 是 |
| `limitation` | 局限/风险 | 段落（醒目提示，⚠ 强调谨慎） | 是 |
| `formula` | 计算原理 | 段落（指标类） | 否 |

渲染器对可选字段"有则渲染、无则跳过"。

**板块定义 `categories.js`**：

```js
{ id:"candlestick", name:"K线基础", icon:"📊", enabled:true,  order:1 }
{ id:"indicator",   name:"技术指标", icon:"📈", enabled:true,  order:2 }
{ id:"pattern",     name:"经典形态", enabled:false, order:3 }
{ id:"theory",      name:"趋势理论", enabled:false, order:4 }
{ id:"intraday",    name:"分时图分析", enabled:false, order:5 }
{ id:"strategy",    name:"实战框架", enabled:false, order:6 }
```

## 5. 图表层

**两类组件：**

1. **`CandleChart`** — 通用 K线绘制
   - 输入：K线数据 `[{o,h,l,c}, ...]` + 可选标注（高亮第 N 根、区域框、箭头、文字）
   - 自动按红涨绿跌上色、绘制影线/实体、网格线
   - 所有 K线形态条目复用，仅喂不同数据 + 标注

2. **`indicators/`** — 指标专用绘制
   - MACD（红绿柱 + DIF/DEA 双线）、KDJ（三线）、MA（叠加在 K线主图上）
   - 复用 `CandleChart` 作为主图，指标画在下方副图或叠加
   - 含指标**计算函数**（MA / MACD / KDJ），与绘制分离，便于单测

**交互**（第一版克制、聚焦教学）：
- 悬停 K线 → 显示该根开/高/低/收数值
- 关键形态"分步高亮"：按钮逐步点亮（如 ①下跌阴线 → ②阳线吞没 → ③确认信号），配文字讲解
- 指标页：开关切换显示/隐藏某条均线或信号点

**注册表 `chartRegistry`**：`chartId` → 图表组件 + 预置数据。`TopicPage` 拿 topic 的 `chartId` 即可渲染，内容与图表解耦。

示意数据手工设计为"标准教科书形态"，比真实数据更干净、更能体现形态特征。

## 6. 页面与导航布局

**① 首页 `/`**
- 顶部：Logo + 标题「技术分析图谱」+ 一句话定位
- 主区：6 个板块卡片，启用的（K线基础、技术指标）可点击，其余 4 个灰色虚线"敬请期待"占位（展示全貌）
- 底部免责声明

**② 板块目录页 `/:category`**（如 `/candlestick`）
- 面包屑 + 板块标题
- 该板块条目卡片网格：深色圆角卡片，含分类标签胶囊 + 迷你 K线示意图 + 名称

**③ 条目详情页 `/:category/:topic`**（核心页面）
```
┌─────────────────────────────────────────┐
│ 看涨吞没  [反转形态][看涨][组合形态]        │  标题 + 标签胶囊
├─────────────────────────────────────────┤
│   ┌───────────────────┐                 │
│   │  可交互 K线示意图   │  分步高亮按钮     │  图表区（含标注，如"吞没区"）
│   └───────────────────┘                 │
├─────────────────────────────────────────┤
│ 含义        段落讲解                       │
│ 识别要点    • ... • ... • ...             │  结构化正文
│ 使用提示    • ... • ... • ...             │
│ 局限/风险   ⚠ 段落（醒目提示）             │
├─────────────────────────────────────────┤
│ 延伸阅读：看跌吞没 · 锤子线                 │  related 跳转
└─────────────────────────────────────────┘
```
- 桌面端：左侧同板块条目列表，便于快速跳转

**导航**：
- 桌面端：顶部导航栏（板块切换）+ 详情页左侧条目列表
- 移动端：响应式，顶栏收为汉堡菜单、侧栏变抽屉；保证手机舒适阅读

**全局视觉**：深色主题（深蓝黑底 `#0b0f1a` 系）、靛蓝主色（`#6366f1` 系）、圆角卡片、标签胶囊、**红涨绿跌**（红 `#f43f5e` 系 / 绿 `#22c55e` 系）。

**免责声明**：页脚固定「本站内容仅供学习参考，不构成任何投资建议」。

## 8. 测试策略（TDD）

- **内容数据层**：校验每个 topic 对象字段完整；`chartId` 在注册表中存在；`category` 合法；`related` 指向真实存在的条目。
- **图表计算**：`CandleChart` 给定数据正确算出涨跌色、影线/实体坐标；指标计算函数（MA / MACD / KDJ）数值正确。
- **渲染**：`TopicPage` 给定 topic 渲染出各结构化区块；可选字段缺失时不报错。
- **路由**：非法 category/topic 显示友好的"未找到"页。
